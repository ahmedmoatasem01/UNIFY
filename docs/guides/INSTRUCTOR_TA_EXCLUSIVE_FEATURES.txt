FR-IT-1: GRADEBOOK MANAGEMENT SYSTEM
================================================================================

Description:
Comprehensive gradebook system for Instructors and TAs to manage student grades,
assignments, and course performance. Only accessible to Instructors and TAs.

Features:
1. Grade Entry and Management
   - Enter grades for assignments, quizzes, exams, projects
   - Support for different assignment types with weight-based calculations
   - Bulk grade entry (import from CSV/Excel)
   - Grade modification with history tracking and audit log
   - Curve grades for entire class or specific assignments

2. Assignment Management
   - Create assignments with due dates, max scores, and weights
   - Set assignment categories (homework, quiz, exam, project, participation)
   - Define grading rubrics for each assignment
   - Track assignment submission status
   - Send reminders for missing submissions

3. Grade Calculation
   - Automatic weighted grade calculation per student
   - Real-time course grade updates as grades are entered
   - Support for dropped lowest scores
   - Extra credit assignment handling
   - Final grade calculation with customizable formulas

4. Grade Analytics
   - Class average and distribution charts
   - Grade trends over time
   - Identify struggling students automatically
   - Compare performance across different sections
   - Export grade reports (CSV, PDF)

5. Student Performance Dashboard
   - View individual student grade breakdowns
   - Track student progress throughout semester
   - Identify students at risk of failing
   - Generate performance reports for advisors
   - Historical grade comparison

Implementation Requirements:

Database Schema:
  CREATE TABLE Assignment (
      Assignment_ID INT IDENTITY(1,1) PRIMARY KEY,
      Course_ID INT NOT NULL,
      Instructor_ID INT NOT NULL,
      Assignment_Name NVARCHAR(255) NOT NULL,
      Assignment_Type NVARCHAR(50),  -- 'homework', 'quiz', 'exam', 'project', 'participation'
      Max_Score DECIMAL(5,2) DEFAULT 100,
      Weight DECIMAL(5,2) DEFAULT 1.0,
      Due_Date DATETIME,
      Created_At DATETIME DEFAULT GETDATE(),
      Drop_Lowest INT DEFAULT 0,  -- Number of lowest scores to drop
      Is_Extra_Credit BIT DEFAULT 0,
      FOREIGN KEY (Course_ID) REFERENCES Course(Course_ID),
      FOREIGN KEY (Instructor_ID) REFERENCES Instructor(Instructor_ID)
  );

  CREATE TABLE Grade (
      Grade_ID INT IDENTITY(1,1) PRIMARY KEY,
      Assignment_ID INT NOT NULL,
      Student_ID INT NOT NULL,
      Score DECIMAL(5,2),
      Max_Score DECIMAL(5,2),
      Percentage DECIMAL(5,2),
      Letter_Grade NVARCHAR(5),
      Graded_By INT,  -- Instructor_ID or TA_ID
      Graded_At DATETIME DEFAULT GETDATE(),
      Notes NVARCHAR(MAX),
      Is_Dropped BIT DEFAULT 0,
      FOREIGN KEY (Assignment_ID) REFERENCES Assignment(Assignment_ID),
      FOREIGN KEY (Student_ID) REFERENCES Student(Student_ID),
      FOREIGN KEY (Graded_By) REFERENCES Instructor(Instructor_ID)
  );

  CREATE TABLE GradeHistory (
      History_ID INT IDENTITY(1,1) PRIMARY KEY,
      Grade_ID INT NOT NULL,
      Old_Score DECIMAL(5,2),
      New_Score DECIMAL(5,2),
      Changed_By INT,  -- Instructor_ID
      Changed_At DATETIME DEFAULT GETDATE(),
      Reason NVARCHAR(500),
      FOREIGN KEY (Grade_ID) REFERENCES Grade(Grade_ID)
  );

FR-IT-2: STUDENT ATTENDANCE TRACKING SYSTEM
================================================================================

Description:
Comprehensive attendance tracking system for Instructors and TAs to record and
monitor student class attendance. Only accessible to Instructors and TAs.

Features:
1. Attendance Recording
   - Mark attendance for each class session
   - Support for: Present, Absent, Late, Excused, Tardy
   - Quick attendance entry (checkboxes for all students)
   - QR code check-in system (future enhancement)
   - Bulk attendance entry for multiple sessions

2. Attendance Analytics
   - Calculate attendance percentage per student
   - Overall class attendance statistics
   - Attendance trends over time (charts/graphs)
   - Identify students with low attendance automatically
   - Compare attendance across different courses

3. Automated Alerts
   - Low attendance warnings (<75%) sent to students
   - Critical attendance alerts (<50%) sent to students and advisors
   - Weekly attendance summary reports
   - Email notifications for attendance issues

4. Attendance Reports
   - Individual student attendance reports
   - Class attendance summary (for instructors)
   - Export attendance data (CSV/PDF)
   - Attendance history with detailed logs
   - Semester attendance overview

5. Integration Features
   - Automatic attendance creation from course schedule
   - Calendar integration (mark attendance on class dates)
   - Link to student profiles
   - Export for administrative purposes

Implementation Requirements:

Database Schema:
  CREATE TABLE AttendanceSession (
      Session_ID INT IDENTITY(1,1) PRIMARY KEY,
      Course_ID INT NOT NULL,
      Session_Date DATE NOT NULL,
      Session_Type NVARCHAR(50),  -- 'lecture', 'lab', 'tutorial'
      Created_By INT,  -- Instructor_ID or TA_ID
      Created_At DATETIME DEFAULT GETDATE(),
      FOREIGN KEY (Course_ID) REFERENCES Course(Course_ID),
      FOREIGN KEY (Created_By) REFERENCES Instructor(Instructor_ID),
      UNIQUE (Course_ID, Session_Date, Session_Type)
  );

  CREATE TABLE Attendance (
      Attendance_ID INT IDENTITY(1,1) PRIMARY KEY,
      Session_ID INT NOT NULL,
      Student_ID INT NOT NULL,
      Status NVARCHAR(20) CHECK (Status IN ('present', 'absent', 'late', 'excused', 'tardy')) DEFAULT 'absent',
      Notes NVARCHAR(500),
      Marked_By INT,  -- Instructor_ID or TA_ID
      Marked_At DATETIME DEFAULT GETDATE(),
      FOREIGN KEY (Session_ID) REFERENCES AttendanceSession(Session_ID),
      FOREIGN KEY (Student_ID) REFERENCES Student(Student_ID),
      FOREIGN KEY (Marked_By) REFERENCES Instructor(Instructor_ID),
      UNIQUE (Session_ID, Student_ID)
  );

FR-IT-3: COURSE CONTENT MANAGEMENT SYSTEM
================================================================================

Description:
System for Instructors and TAs to manage course materials, lecture notes,
syllabus, and resources. Only accessible to Instructors and TAs.

Features:
1. Course Materials Management
   - Upload course materials (PDFs, documents, videos, links)
   - Organize materials by week/topic/module
   - Create folders and categories
   - Set material visibility (visible to all students, specific students, or hidden)
   - Version control for updated materials

2. Syllabus Management
   - Create and edit course syllabus
   - Define course objectives and learning outcomes
   - Set grading policies and weights
   - Define course schedule and important dates
   - Publish syllabus to students

3. Lecture Notes Management
   - Upload lecture slides and notes
   - Link notes to specific class sessions
   - Organize by topic or week
   - Allow students to download (if enabled)
   - Track download statistics

4. Resource Library
   - Maintain library of reference materials
   - Link to external resources (textbooks, websites, videos)
   - Categorize resources by topic
   - Recommend resources to specific students
   - Track resource usage

5. Announcements and Updates
   - Post course announcements
   - Send updates to all enrolled students
   - Schedule announcements for future posting
   - Archive old announcements
   - Track announcement read status

Implementation Requirements:

Database Schema:
  CREATE TABLE CourseMaterial (
      Material_ID INT IDENTITY(1,1) PRIMARY KEY,
      Course_ID INT NOT NULL,
      Material_Name NVARCHAR(255) NOT NULL,
      Material_Type NVARCHAR(50),  -- 'syllabus', 'lecture', 'assignment', 'resource', 'announcement'
      File_Path NVARCHAR(500),
      File_Name NVARCHAR(255),
      File_Size BIGINT,
      Category NVARCHAR(100),
      Week_Number INT,
      Topic NVARCHAR(255),
      Is_Visible BIT DEFAULT 1,
      Created_By INT,  -- Instructor_ID or TA_ID
      Created_At DATETIME DEFAULT GETDATE(),
      Updated_At DATETIME,
      FOREIGN KEY (Course_ID) REFERENCES Course(Course_ID),
      FOREIGN KEY (Created_By) REFERENCES Instructor(Instructor_ID)
  );

  CREATE TABLE CourseAnnouncement (
      Announcement_ID INT IDENTITY(1,1) PRIMARY KEY,
      Course_ID INT NOT NULL,
      Title NVARCHAR(255) NOT NULL,
      Content NVARCHAR(MAX) NOT NULL,
      Priority NVARCHAR(20) CHECK (Priority IN ('low', 'medium', 'high', 'urgent')) DEFAULT 'medium',
      Is_Pinned BIT DEFAULT 0,
      Scheduled_Date DATETIME,
      Created_By INT,  -- Instructor_ID or TA_ID
      Created_At DATETIME DEFAULT GETDATE(),
      FOREIGN KEY (Course_ID) REFERENCES Course(Course_ID),
      FOREIGN KEY (Created_By) REFERENCES Instructor(Instructor_ID)
  );

  CREATE TABLE MaterialAccess (
      Access_ID INT IDENTITY(1,1) PRIMARY KEY,
      Material_ID INT NOT NULL,
      Student_ID INT NOT NULL,
      Accessed_At DATETIME DEFAULT GETDATE(),
      Download_Count INT DEFAULT 0,
      FOREIGN KEY (Material_ID) REFERENCES CourseMaterial(Material_ID),
      FOREIGN KEY (Student_ID) REFERENCES Student(Student_ID)
  );

FR-IT-4: STUDENT PROGRESS MONITORING DASHBOARD
================================================================================

Description:
Comprehensive dashboard for Instructors and TAs to monitor student progress,
performance, and engagement across their courses. Only accessible to Instructors and TAs.

Features:
1. Student Performance Overview
   - View all students in course with current grades
   - Sort and filter students by performance, attendance, or engagement
   - Identify at-risk students automatically
   - Compare student performance to class average
   - Track student progress over time

2. Engagement Metrics
   - Assignment submission rates per student
   - Participation scores (if tracked)
   - Login frequency and activity
   - Material access statistics
   - Response time to messages/announcements

3. Intervention Tools
   - Flag students for additional support
   - Send personalized messages to struggling students
   - Schedule office hours appointments
   - Create study group recommendations
   - Generate intervention reports

4. Class Analytics
   - Overall class performance statistics
   - Grade distribution charts
   - Assignment completion rates
   - Attendance trends
   - Engagement heatmaps

5. Comparative Analysis
   - Compare performance across different sections
   - Historical comparison (same course previous semesters)
   - Identify course difficulty issues
   - Track improvement after interventions

Implementation Requirements:

Database Schema:
  CREATE TABLE StudentProgress (
      Progress_ID INT IDENTITY(1,1) PRIMARY KEY,
      Student_ID INT NOT NULL,
      Course_ID INT NOT NULL,
      Current_Grade DECIMAL(5,2),
      Attendance_Percentage DECIMAL(5,2),

      UNIQUE (Student_ID, Course_ID)
  );

  CREATE TABLE Intervention (
      Intervention_ID INT IDENTITY(1,1) PRIMARY KEY,
      Student_ID INT NOT NULL,
      Course_ID INT NOT NULL,
      Intervention_Type NVARCHAR(50),  -- 'message', 'office_hours', 'study_group', 'tutoring'

      Scheduled_Date DATETIME,
      Notes NVARCHAR(MAX),
      Created_By INT,  -- Instructor_ID or TA_ID
      Created_At DATETIME DEFAULT GETDATE(),
      FOREIGN KEY (Student_ID) REFERENCES Student(Student_ID),
      FOREIGN KEY (Course_ID) REFERENCES Course(Course_ID),
      FOREIGN KEY (Created_By) REFERENCES Instructor(Instructor_ID)

IMPLEMENTATION PRIORITY
================================================================================

Recommended implementation order based on value and dependencies:

1. FR-IT-1: Gradebook Management System (High Priority)
   - Core functionality for instructors
   - Essential for academic management
   - Integrates with existing enrollment system
   - High daily usage value

2. FR-IT-2: Student Attendance Tracking (High Priority)
   - Important for course management
   - Relatively straightforward implementation
   - High instructor value
   - Complements gradebook system

3. FR-IT-4: Student Progress Monitoring Dashboard (Medium Priority)
   - Valuable for identifying at-risk students
   - Integrates with gradebook and attendance
   - Helps with proactive intervention
   - Moderate complexity

4. FR-IT-3: Course Content Management (Medium Priority)
   - Useful for organizing course materials
   - Requires file upload infrastructure
